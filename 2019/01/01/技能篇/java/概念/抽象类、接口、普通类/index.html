抽象类、接口、普通类

发表于 2019-01-01 更新于 2020-09-05 分类于 技能篇 ， java ， 概念
本文字数： 1.6k 阅读时长 ≈ 1 分钟
  • 抽象类就像是一个模版，各个子类按照抽象类进行实现，可以抽象类的方法，也可以添加自己的方法。
  • 抽象类使用来捕捉子类的抽象方法，它不能被实例化。只能作为子类地超类，抽象类是被用来创建继承层级里子类的模版。
  • 接口是抽象方法的集合。如果一个类实现了某个接口，那么他就继承了这个接口的抽象方法，这就像契约模式，如果实现了这个接口，就必须确保诗这些方法。接口只是一种形式，接口本身不能实现任何事情。

抽象类

  • 抽象方法：抽象方法只有声明，没有具体的实现。接口中的方法都是抽象方法。抽象方法必须要有abstract方法进行修饰。如果一个类含有抽象方法，则称这个类为抽象类，抽象类前必须也要用abstract进行修饰。由于抽象类中含有无具体实现的方法（抽象方法）。所以抽象类不能通过new方法来创建对象，也不能利用java反射进行创建对象。
  • 抽象类的定义是包含抽象方法的类，但是抽象类不一定必须包含抽象方法。
  • 抽象类是为了继承存在的，因为抽象类不能自己去创建对象。所以通过子类去继承抽象类的方法来使用抽象类。
  • 对于一个父类，如果一个方法在父类中实现没有任何意义，必须根据子类的实际需求来进行不同的实现，那么就可以将这个方法声明为抽象方法，这个类也即抽象类。
  • 包含抽象方法的类是抽象类，但是抽象类不只包含抽象方法，同样也可以拥有成员变量和普通的成员方法。

接口：

  • 借口泛指供别人调用的方法或函数。接口是对行为的抽象。
  • 接口类中可以含有自己的变量和方法，但是需要注意，接口中的变量会被隐式的指定为public static final变量（并且只能是public static final变量，用private修饰会报编译错误），接口中的方法会被隐式的指定为public abstract方法（其他关键字，植入private，protected、static final等修饰都会报编译错误）。并且接口中的方法都不能有具体的实现。接口中的所有方法都必须是抽象方法。
  • 接口是一种极度的抽象，比抽象类更抽象的类。一般情况下也不在接口中定义变量。
  • 一个类允许实现多个接口，如果一个类实现了某个接口，就必须实现这个接口中的所有方法，对于遵循某个接口的抽象类，可以不实现接口中的抽象方法。在子类继承抽象类的时候自然会实现该方法。

抽象类和普通类的区别

  1. 抽象类的类权限必须诗public或protected（如果是private的话，这个类就不能被子类继承，这个抽象类就没有意义）。抽象类的默认类型是public。
  2. 抽象类不能用来创建一个新的对象。
  3. 如果一个子类继承抽象类，则这个子类必须实现抽象类中的抽象方法。如果子类没有没有实现父类的方法，则子类也必须是抽象方法。

抽象类和接口的区别

语法层次的区别：

  1. 抽象类提供的成员方法可以由具体的实现，而接口中的方法都是抽象方法，不能有具体的实现。
  2. 抽象类中的成员变量可以是各种类型的,而接口中的成员变量只能是public static final类型的。
  3. 接口中不能有静态代码块以及静态方法，而抽象类中可以都可以有。
  4. 一个类只能继承一个抽象类，但是可以实现多个接口。

使用的时候的区别：

  • 如果在一项功能中由一些方法希望这些方法默认实现，那么可以使用抽象类。抽象类可以不需要在子类中再实现可以直接调用子类的方法。
  • 在java中由于不支持多重继承，子类不能继承多个类。所以在java中如果要实现多重继承，只能通过接口来实现，可以实现多个接口来解决多重继承的问题。但是需要在子类中实现通用的方法，且子类必须实现所有接口的方法。而所有的接口都是public的，对于子类的安全性无法保证。

使用接口的缺点

  1. 如果基础功能在不断的变化，那么就需要使用抽象类。如果是通过接口实现的话，就需要修改所有实现类的中实现这个接口的方法。
  2. 接口的实现是稍微慢（相对于抽象类），他需要时间去寻找在类中实现的方法。
