注意事项

  1. 如果以同一个字段既分库又分表，则需要保证分库和分表的策略（取模）互为质数。否则分库分表之后的数据存储会不平均，会导致有某些数据节点没有数据。
  2. 分库分表的时候最好不要使用联合查询，如果需要联合查询最好配置绑定表
  3. 分库分表的表最好有统一的一个字段，这样能够保证同一个用户的数据存储在同一个数据库中，且存储数据的真实表的后缀也一样，便于后续执行DBA的操作。
  4. springboot2.x的application.yml中需要添加spring.main.allow-bean-definition-overriding=true的配置

issue2:sharding-sphere不支持mysql的select last_insert_id()函数

sharding-sphere不支持mysql的select last_insert_id()函数，当需要获取插入数据的id的时候，就无法通过这个sql获取，所以需要自己添加事务，并重新查一下数据库获取id（查询的时候最好加上where条件，将范围缩小，避免出现问题）

issue3:sharding-sphere(3.1.x)不支持limit offset limit语法。

sharding-sphere(3.1.x)不支持limit offset limit语法，解析之后的limit offset limit===>limit 0,offset+limit，这会导致从offet获取数据的时候获取的是从0开始到offset+limit的所有数据。

解决办法：

  1. 将sharing-sphere的版本降到3.0.0，这个版本对这个sql语法是支持的，但是降到3.0.0之后无法使用分布式事务
  2. 等待sharing-sphere升级到4.0.0版本(预计2019年11月份发布)
