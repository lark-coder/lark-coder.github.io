float类型在Java中占用4字节，long类型在 中占用8字节，为什么float类型的取值范围比long类型的取值范围还大？

java中long类型占用8个字节，取值范围是-2^632^63-1，大约是9*10^18。而float类型占用4个字节，取值范围是-3.403E383.403E38。
long类型是整数类型，在内存中占用8个字节，共64位，它表示的数值有2^64方个，平分正负之后取值范围就是-2^63~2^63-1。
float类型在内存中占用4个字节，共32位，但是float在内存中的存储和整数类型不一样，它的公式是：V=(-1)^s * M * 2^E，其中第一位是s，表示正负号，第二位到第九位是M表示指数域，即E；第十位到第三十二位表示小数域，即M的取值范围为[1，2）或[0，1）。
浮点数在内存中的二进制值不是直接转换为十进制数值的，而是按照上述公式计算而来，通过这个公式，虽然只用到了4个字节，但是浮点数却比长整型的最大值要大。
Long是整形，long是精确的，精确到 个位 的。而Float是浮点型，它是有有效位数的.

使用“+”可以连接两个字符串（String对象），那么，是怎样进行连接的？

java中string的“+”操作实际执行的是Stringbuilder的操作，而且在执行的时候每个+都会重新new 一个stringbuilder，但是java虚拟机也对“+”进行了优化。普通的字符串拼接使用“+”没有问题。但是如果大批量的字符串拼接，最好使用stringBuilder，如果能够指定stringbuilder的容量就最好了，stringbuilder的容量初始是16，会进行扩容（扩容机制是翻倍，而且会发生数据的复制，因为stringbuilder的后台实现是char[]数组）。如果需要线程安全则需要使用stringbuffer。如果知识两个字符串的拼接使用concat性能是最好的，因为concat调用了一次Arrays.copyOf，并且指定了len + otherLen，相当于分配了一次内存空间，并分别从str1和str2各复制一次数据。而如果使用StringBuilder并指定capacity，相当于分配一次内存空间，并分别从str1和str2各复制一次数据，最后因为调用了toString方法，又复制了一次数据。
所有的“+”、stringbuilder、stringbuffer最后都会执行tostring方法。

java的==和equals的区别

发表于 2019-01-01 更新于 2020-09-05 分类于 技能篇 ， java ， 概念
本文字数： 1.6k 阅读时长 ≈ 1 分钟

java的==和equals的区别：

  • java中的==生成的一个boolean类型的值，比较的是两个对象的值是否相等。
  • 对于java中的基本数据类型（byte（1bit）、char（2bit）、int（4bit）、long(8bit)、short(2bit)、float(2bit)、double(4bit)、boolean（1bit）和非基本数据类型（string），==的处理逻辑并不相同。
  • 非基本数据类型（引用数据类型）存储的并不是实际的值，而是对象在内存中的地址。所以==比较的就是内存中地址，而不是实际的值。

String的equals方法

String s1 = “123”;
String s2 = “123”;
这两个String的==对比是True，因为java对String做了单独的处理，java维护了一个String Pool，用来存放运行中存储的各种字符串。并且池中的内容不重复，一般的其他对象不存在这个缓存池中，并且创建的对象仅仅存在于方法的堆栈区。

抽象类、接口、普通类

发表于 2019-01-01 更新于 2020-09-05 分类于 技能篇 ， java ， 概念
本文字数： 1.6k 阅读时长 ≈ 1 分钟
  • 抽象类就像是一个模版，各个子类按照抽象类进行实现，可以抽象类的方法，也可以添加自己的方法。
  • 抽象类使用来捕捉子类的抽象方法，它不能被实例化。只能作为子类地超类，抽象类是被用来创建继承层级里子类的模版。
  • 接口是抽象方法的集合。如果一个类实现了某个接口，那么他就继承了这个接口的抽象方法，这就像契约模式，如果实现了这个接口，就必须确保诗这些方法。接口只是一种形式，接口本身不能实现任何事情。

抽象类

  • 抽象方法：抽象方法只有声明，没有具体的实现。接口中的方法都是抽象方法。抽象方法必须要有abstract方法进行修饰。如果一个类含有抽象方法，则称这个类为抽象类，抽象类前必须也要用abstract进行修饰。由于抽象类中含有无具体实现的方法（抽象方法）。所以抽象类不能通过new方法来创建对象，也不能利用java反射进行创建对象。
  • 抽象类的定义是包含抽象方法的类，但是抽象类不一定必须包含抽象方法。
  • 抽象类是为了继承存在的，因为抽象类不能自己去创建对象。所以通过子类去继承抽象类的方法来使用抽象类。
  • 对于一个父类，如果一个方法在父类中实现没有任何意义，必须根据子类的实际需求来进行不同的实现，那么就可以将这个方法声明为抽象方法，这个类也即抽象类。
  • 包含抽象方法的类是抽象类，但是抽象类不只包含抽象方法，同样也可以拥有成员变量和普通的成员方法。

接口：

  • 借口泛指供别人调用的方法或函数。接口是对行为的抽象。
  • 接口类中可以含有自己的变量和方法，但是需要注意，接口中的变量会被隐式的指定为public static final变量（并且只能是public static final变量，用private修饰会报编译错误），接口中的方法会被隐式的指定为public abstract方法（其他关键字，植入private，protected、static final等修饰都会报编译错误）。并且接口中的方法都不能有具体的实现。接口中的所有方法都必须是抽象方法。
  • 接口是一种极度的抽象，比抽象类更抽象的类。一般情况下也不在接口中定义变量。
  • 一个类允许实现多个接口，如果一个类实现了某个接口，就必须实现这个接口中的所有方法，对于遵循某个接口的抽象类，可以不实现接口中的抽象方法。在子类继承抽象类的时候自然会实现该方法。

对Java 平台的理解

发表于 2019-01-01 更新于 2020-09-05 分类于 技能篇 ， java ， 概念
本文字数： 1.2k 阅读时长 ≈ 1 分钟

经典回答

  • Java语言是一种面向对象的高级语言，它最显著的有两个特性，一是通过平台中立的class文件格式和屏蔽底层硬件差异的jvm实现‘一次编写，到处运行’；二是通过‘垃圾收集器’管理内存的分配和回收。
  • Java 是一种简单、严谨并且适合编写的语言，它不像C/C++ 那样有很多晦涩难懂的内容，如头文件、指针、结构等等。我们编写java 会接触到 JRE（Java Runtime Environment）或者 JDK（Java Development Kit）。 JRE，也就是 Java 运行环境，包含了JVM 和丰富的 Java 类库，比如集合，泛型，反射，并发，网络，IO/NIO等。而 JDK 可以作是 JRE 的一个超集，提供了更多工具，比如编译器、各种诊断工具等，让java 语言更加安全、健壮。还有更多第三方类库的支持，如spring、springmvc、mybatis等让java 非常适合开发。
  • 很多对Java 的影响是它很慢，因为它是解释执行。其实不是这样的，Java 是解释执行+编译执行混合型。一般JVM 是直接解释执行.class文件，当出现高频代码时JVM 会通过JIT 即时编译器把经常运行的代码作为”热点代码”编译与本地平台相关的机器码，并进行各种层次的优化。

知识点

  • Java是纯面向对象的语言。《Java编程思想》中提到Java语言是一种“Everything is object”的语言，它能够直接反映现实生活中的对象，例如车、动物等，因此通过它编写程序更容易。
  • 平台无关性。Java语言可以“一次编译，到处执行”。无论是在Windows平台还是在Linux、MacOS等其他平台上对Java程序进行编译，编译后的程序在其他平台上都可以运行。由于Java是解释性语言，编译器会将Java代码变成“中间代码”，然后在Java虚拟机（Java Virtual Machine，JVM）上解释执行。由于中间代码与平台无关，因此Java语言可以很好的跨平台执行，具有很好的可移植性。
  • Java提供了很多内置的类库，通过这些类库，简化了开发人员的程序设计工作，同时缩短了项目的开发时间，例如，Java语言提供了对多线程的支持，提供了对网络通信的支持，最主要的是提供了垃圾回收器，这使得开发人员从内存的管理中解脱出来。
  • 提供了对Web应用开发的支持。例如，Applet、Servlet和JSP可以用来开发Web应用程序；Socket、RMI可以用来开发分布式应用程序。
  • 具有较好的安全性和健壮性。Java语言经常被用在网络环境中，为了增强程序的安全性，Java语言提供了一个防止恶意代码攻击的安全机制（数组边界检测和Bytecode校验等）。Java的强类型机制、垃圾回收器、异常处理和安全检查机制使得用Java语言编写的程序具有很好的健壮性。
  • 去除了C++语言中难以理解、容易混淆的特性，如头文件、指针、结构、单元、运算符重载、虚拟基础类、多重继承等，使得程序更加严谨简洁。

HashTable, HashMap，TreeMap区别

  • HashTable线程同步，HashMap非线程同步
  • HashTable不允许<键，值>有空值。HashMap允许<键，值>有空值。
  • HashTable使用的是Enummeration，HashMap使用的是Iterator。
  • HashTable中中的hash数组的默认大小是11，增加方式是old*2+1,HashMap中的Hash数组的默认大小是16，增长方式一定是2的指数倍。
  • TreeMap能够把它保存的记录根据键排序，默认是按升序排序

Vector,ArrayList, LinkedList的区别

  • Vector、ArrayList都是以类似数组的形式存储在内存中，LinkedList则以链表的形式进行存储。
  • List中的元素有序、允许重复的元素。Set中的元素无序，不允许有重复的元素。
  • Vector线程同步，ArrayList和LinkedList不同步
  • LinkedList适合在指定位置插入、删除操作，不适合查找；ArrayList、Vector适合查找，不适合指定位置的插入，删除操作。
  • ArrayList在元素填满容器时会自动扩充容器大小的50%，而Vector则是100%，因此ArrayList更节省空间

Java集合相关问题

本文字数： 4.7k 阅读时长 ≈ 4 分钟

Set的底层：

  1. HashSet的性能总是比TreeSet好(特别是最常用的添加、查询元素等操作)，因为TreeSet需要额外的红黑树算法来维护集合元素的次序。只有当需要一个保持排序的Set时，才应该使用TreeSet，否则都应该使用HashSet
  2. 对于普通的插入、删除操作，LinkedHashSet比HashSet要略慢一点，这是由维护链表所带来的开销造成的。不过，因为有了链表的存在，遍历LinkedHashSet会更快
  3. EnumSet是所有Set实现类中性能最好的，但它只能保存同一个枚举类的枚举值作为集合元素
  4. HashSet、TreeSet、EnumSet都是”线程不安全”的，通常可以通过Collections工具类的synchronizedSortedSet方法来”包装”该Set集合。
    SortedSet s = Collections.synchronizedSortedSet(new TreeSet(…));
