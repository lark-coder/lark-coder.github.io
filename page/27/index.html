String、StringBuffer、StringBuilder的区别

本文字数： 614 阅读时长 ≈ 1 分钟

String、StringBuffer、StringBuilder的区别：

运行速度：StringBuiler>StringBuffer>String
String最慢的原因是String是字符串常量，而StringBuffer和StringBuilder是字符串变量，即String对象一旦创建之后该对象时不可更改的，但后两者的对象时变量，是可以更改的。对变量的操作其实就是对对象的修改，而不用创建和回收的操作，所以速度比String快。

  • String是字符串常量，StringBuffer和StringBuilder都是字符串变量，后两者的字符内容可变，而前者创建后的内容不可变
  • String不可变的原因是在JDK中String类被声明为一个final类
  • StringBuffer是线程安全的，而StringBuilder是非线程安全的
  • 线程安全会带来额外的系统开销，所以StringBuilder的效率是最高的。如果对系统中的线程是否安全无法掌握，可以使用StringBuffer，在线程不安全的地方加上关键字Synchronized。
  • String、StringBuilder、StringBuffer的实际区别可以通过对字符串的值进行修改，然后观察hashCode的值判断，String的hashCode在值发生变化之后的hashCode也发生变化。而StringBuilder和StringBuffer则不回发生变化。

java多线程

多线程是在同一个程序内部并行执行，因此会对相同的内存空间进行并发读写操作。

多线程的优点

  • 资源利用率更好
  • 程序设计在某些情况下更简单
  • 程序响应更快。

    多线程的缺点

  • 设计更加复杂
  • 上下文切换（当CPU从执行一个线程切换到另一个线程的euihz，它需要先存储当前线程的本地的数据、程序指针等，然后载入另一个线程的本地数据、程序指针等，最后才开始执行。CPU会在一个上下文中执行一个线程，然后切换到另外一个上下文中执行另外一个线程）的开销
  • 增加资源消耗

java线程池是什么？

所谓的线程池，就是将多个线程放在一个池子里面（池技术）。然后需要线程的时候不是创建一个线程，而是从线程池里面获取一个可用的线程，然后执行我们的任务。线程池的关键在于它为我们管理了多个线程，我们不需要关心如何创建线程，我们只需要关心我们的业务，然后需要线程来执行任务的时候从线程池中获取线程。线程执行完之后线程不回被销毁。而是重新放回到池子里面，等待机会去执行任务。

为什么需要线程池

首先一点就是线程池为我们提供了一中简易的多线程变成方案，我们不需要投入太多的经理去管理多个线程，线程池回自动帮我们管理好，它什么时候该做什么事情，我们只要再需要的时候去获取就可以了。
其次，我们使用线程池很大程度上归咎与创建和销毁线程的待见是昂贵的，甚至我们创建和销毁线程的资源比我们实际执行任务的所花费的时间都要长，这是不合理的，如果没有一个合理的管理者，可能会出现创建了过多线程的情况。也就是jvm中活跃的线程过多，过多的线程可能会导致线程过度切换的问题。

互斥锁

在多线程环境中往往存在因某一资源被同时访问导致该资源不一致的问题，互斥锁 通过排它性，即同时只允许一个访问者对其进行访问来保证资源的有效同步，但它无法限制线程对该资源的访问顺序，因此线程对资源的访问也是无序的。

自旋锁

在互斥锁中，如果线程 A 在请求锁的时候发现该锁已被线程 B 霸占，那么此时线程 A 便会进入休眠状态，直到锁被线程 B 释放后被系统唤醒。但在自旋锁中，线程 A 在发现锁被霸占时并不进入休眠，而是一直循环查看锁的持有者是否已经释放了该锁。初看起来，这种霸占CPU资源的做法极其低效，但与互斥锁仔细对比后我们可以发现它有以下几个优点：

Error和Exception

发表于 2019-01-01 更新于 2020-09-05 分类于 技能篇 ， java ， 异常

1. Error 和 Exception 区别是什么?

  • Error 类型的错误通常为虚拟机相关错误，如系统崩溃，内存不足，堆栈溢出等，编译器不会对这类错误进行检测，JAVA 应用程序也不应对这类错误进行捕获，一旦这类错误发生，通常应用程序会被终止，仅靠应用程序本身无法恢复；
  • Exception 类的错误是可以在应用程序中进行捕获并处理的，通常遇到这种错误，应对其进行处理，使应用程序可以继续正常运行。

2. 运行时异常和一般异常区别是什么？

  • 编译器不会对运行时异常进行检测，没有 try-catch，方法签名中也没有 throws 关键字声明，编译依然可以通过。如果出现了 RuntimeException, 那一定是程序员的错误。
  • 一般一场如果没有 try-catch，且方法签名中也没有用 throws 关键字声明可能抛出的异常，则编译无法通过。这类异常通常为应用环境中的错误，即外部错误，非应用程序本身错误，如文件找不到等。

float类型在Java中占用4字节，long类型在 中占用8字节，为什么float类型的取值范围比long类型的取值范围还大？

java中long类型占用8个字节，取值范围是-2^632^63-1，大约是9*10^18。而float类型占用4个字节，取值范围是-3.403E383.403E38。
long类型是整数类型，在内存中占用8个字节，共64位，它表示的数值有2^64方个，平分正负之后取值范围就是-2^63~2^63-1。
float类型在内存中占用4个字节，共32位，但是float在内存中的存储和整数类型不一样，它的公式是：V=(-1)^s * M * 2^E，其中第一位是s，表示正负号，第二位到第九位是M表示指数域，即E；第十位到第三十二位表示小数域，即M的取值范围为[1，2）或[0，1）。
浮点数在内存中的二进制值不是直接转换为十进制数值的，而是按照上述公式计算而来，通过这个公式，虽然只用到了4个字节，但是浮点数却比长整型的最大值要大。
Long是整形，long是精确的，精确到 个位 的。而Float是浮点型，它是有有效位数的.

使用“+”可以连接两个字符串（String对象），那么，是怎样进行连接的？

java中string的“+”操作实际执行的是Stringbuilder的操作，而且在执行的时候每个+都会重新new 一个stringbuilder，但是java虚拟机也对“+”进行了优化。普通的字符串拼接使用“+”没有问题。但是如果大批量的字符串拼接，最好使用stringBuilder，如果能够指定stringbuilder的容量就最好了，stringbuilder的容量初始是16，会进行扩容（扩容机制是翻倍，而且会发生数据的复制，因为stringbuilder的后台实现是char[]数组）。如果需要线程安全则需要使用stringbuffer。如果知识两个字符串的拼接使用concat性能是最好的，因为concat调用了一次Arrays.copyOf，并且指定了len + otherLen，相当于分配了一次内存空间，并分别从str1和str2各复制一次数据。而如果使用StringBuilder并指定capacity，相当于分配一次内存空间，并分别从str1和str2各复制一次数据，最后因为调用了toString方法，又复制了一次数据。
所有的“+”、stringbuilder、stringbuffer最后都会执行tostring方法。
