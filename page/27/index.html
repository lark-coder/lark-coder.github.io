花开无声

第一季度的总结，念起于年初，计划写于三月底，当时却是给忘记了，直到5月初才想起，但是当时想的是第一季度已经过去两个月了，现在再来写这个总结也不是当初的心情和感受了，就又给放弃了。这两三周以来，那个念头越来越强烈，最终还是动笔开始写下这份总结，顺便再回味一下那段心路。

“花开无声”，这个词已经酝酿在脑海之中已经很久很久，用来形容整个第一季度再恰当不过。去年一整年都被工作主宰着，在自己的生活当中自己却没有了立足之地，现在慢慢地走出那段忙碌的时光，再去品味那段时光的苦涩，才渐渐有将生活掌握在手中的感觉。 其实在第一季度想了很多很多，但是太多都被记忆所抛弃，被时间悄无声息地抹去

这次的总结毕竟写于6月中旬，必然很难还原当初的那种心态，也必然被现在的心态包装，但是聊胜于无，总能给第一季度一个交代。接下来准备第二季度的总结，换一种心态，换一种写法，换一种自己。

String、StringBuffer、StringBuilder的区别

发表于 2019-01-01 更新于 2020-09-05 分类于 技能篇 ， java ， String
本文字数： 614 阅读时长 ≈ 1 分钟

String、StringBuffer、StringBuilder的区别：

运行速度：StringBuiler>StringBuffer>String
String最慢的原因是String是字符串常量，而StringBuffer和StringBuilder是字符串变量，即String对象一旦创建之后该对象时不可更改的，但后两者的对象时变量，是可以更改的。对变量的操作其实就是对对象的修改，而不用创建和回收的操作，所以速度比String快。

  • String是字符串常量，StringBuffer和StringBuilder都是字符串变量，后两者的字符内容可变，而前者创建后的内容不可变
  • String不可变的原因是在JDK中String类被声明为一个final类
  • StringBuffer是线程安全的，而StringBuilder是非线程安全的
  • 线程安全会带来额外的系统开销，所以StringBuilder的效率是最高的。如果对系统中的线程是否安全无法掌握，可以使用StringBuffer，在线程不安全的地方加上关键字Synchronized。
  • String、StringBuilder、StringBuffer的实际区别可以通过对字符串的值进行修改，然后观察hashCode的值判断，String的hashCode在值发生变化之后的hashCode也发生变化。而StringBuilder和StringBuffer则不回发生变化。
